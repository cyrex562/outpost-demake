# CLAUDE.md

## Project Mission

This repository is focused on **decompiling, understanding, and rebuilding Outpost (DOS/Windows era)**.

Primary goals:

- Recover readable/maintainable source from decompiler output.
- Reconstruct buildable C/Rust code paths.
- Preserve behavioral parity with the original game.

---

## Key Top-Level Directories

- `decompiled_c/` — main exported C/H source from Ghidra, compatibility headers, and split artifacts.
- `reversing/` — reverse-engineering project assets, including the Ghidra project and Binary Ninja databases.
- `resources/` — original binaries and game asset files extracted from the original CD-ROM.
- `archive/` — historical and intermediary conversion artifacts.
- `scripts/` — automation for analysis, orchestration, and code splitting/indexing.

---

## `decompiled_c/` Structure (Where to Find Things)

- `decompiled_c/outpost.c`
  - Large monolithic Ghidra-exported C source.
- `decompiled_c/outpost.h`
  - Large monolithic exported header with structs/typedefs/prototypes.
- `decompiled_c/ghidra.h`
  - Compatibility macros/types for decompiler-generated code patterns.
- `decompiled_c/CMakeLists.txt`
  - Baseline CMake scaffold for attempting rebuild/compilation experiments.
- `decompiled_c/split_outpost/`
  - LLM-friendly split outputs and indexes generated by `scripts/split_ghidra_exports.py`.
  - Important files:
    - `MANIFEST.md` — first-pass split inventory.
    - `outpost_c/` and `outpost_h/` — size-based chunks.
    - `themed/` — thematic chunks and third-pass indexes:
      - `THEMED_MANIFEST.md`
      - `THEMED_INDEX.json` / `THEMED_INDEX.md`
      - `THEMED_FUNCTION_INDEX.json` / `THEMED_FUNCTION_INDEX.md`
      - `outpost_c/<theme>/...`
      - `outpost_h/<theme>/...`

---

## Reverse-Engineering Source of Truth

The Ghidra project is located under:

- `reversing/ghidra_projects/`

When uncertainty exists in decompiled output, **refer back to Ghidra** for validation. Prefer using the **Ghidra MCP tooling** to:

- inspect decompilation and disassembly at source,
- verify symbols/structures/callers,
- correct assumptions before refactoring or porting.

---

## Original Game Data

`resources/` includes:

- `resources/assets/binaries/` — original executable/binary material,
- `resources/assets/` subfolders — original asset files from the CD-ROM (audio, maps, images, music, UI, shaders, etc.),
- metadata helpers such as `data_addresses.csv` and `ui_control_ids.json`.

Treat these as canonical reference data during rebuild verification.

---

## Recommended Workflow

1. Start with themed/function index files in `decompiled_c/split_outpost/themed/`.
2. Navigate to the relevant chunk file(s) instead of opening monolithic files.
3. Cross-check unclear logic in `reversing/ghidra_projects/` using Ghidra MCP.
4. Apply targeted edits in converted/organized code locations.
5. Keep behavior-aligned notes when translating to cleaner implementations.
